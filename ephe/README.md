# Файлы эфемерид Swiss Ephemeris (`.se1`)

Бинарные файлы Swiss Ephemeris для режима **SWIEPH** (полные файлы, как в
проф. астропроцессорах). Заголовок настоящего файла начинается со слова
`SWISSEPH` — если внутри `<!DOCTYPE html>`, это страница-обёртка, а не файл.

## Что здесь и зачем
- **Ядро** — `sepl_18.se1` (планеты), `semo_18.se1` (Луна), `seas_18.se1`
  (главные астероиды: Хирон, Фолус, Церера, Паллада, Юнона, Веста; Лилит
  считается аналитически). Эти три файла нужны **питон-эталону** (`spike/`) для
  сверки SWIEPH↔SWIEPH. В самом приложении они УЖЕ внутри пакета `swisseph-wasm`
  (`swisseph.data`), отдельно класть в APK не нужно.
- **Доп. объекты / TNO** — кладутся сюда, в подпапки `astN` (N = номер/1000), и
  ВШИВАЮТСЯ в APK (в пакете их нет). На устройстве при старте пишутся в
  виртуальную ФС движка (`Module.FS`), см. `spike/probe_extra_objects.mjs`.

## Набор астролога (карлики/TNO) — РАЗМЕЩЁН (19 объектов)
Файлы прислала астролог (из набора Алоиса), разложены по `astN/` (N=номер/1000):

`ast19` Хаос · `ast20` Варуна · `ast28` Иксион · `ast38` Хуйя · `ast50` Квавар ·
`ast55` 2002 UX25 · `ast84` 2003 VS2 · `ast90` Седна/Оркус/2004 GV9 ·
`ast120` Салация · `ast136` Хаумеа/Эрида/Макемаке · `ast174` Варда ·
`ast208` 2003 AZ84 · `ast225` Гонгун · `ast307` 2002 MS4 · `ast541` Лелеакухонуа.

Все «короткие» (`…s.se1`, охват 1500–2100), кроме Эриды (`s136199.se1`, длинный,
3000 BCE–2999 CE). Имена/коды берутся ДАННЫМИ (скан этой папки) — добавишь новый
файл, он подхватится сам. Имя нужного файла всегда подсказывает библиотека в
тексте ошибки «SwissEph file '…' not found».

Проверка (дамп эталона + сверка SWIEPH↔SWIEPH, 38 объектов):
`cd spike && ../.venv/Scripts/python.exe dump_swieph.py && npm run verify:swieph`
