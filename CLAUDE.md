# CLAUDE.md — память проекта для Claude Code

Локальное Android-приложение для астролога (рабочее имя **Astra**). Оффлайн-первое,
русский интерфейс. Полное ТЗ — в `ЗАДАНИЕ.md`. Этот файл — короткие правила и
контекст, которые Claude Code должен держать в голове постоянно.

## Что строим (одна строка)
Удобное оффлайн-приложение «небо по дням»: аспекты, колесо транзитов, заметки,
напоминания, библиотека трактовок, чат на своём API-ключе.

## Окружение (зафиксировано, проверено)
- ОС разработки: **Windows**.
- Python: **3.12.10** в изолированном `.venv` (НЕ 3.14 — на нём `pyswisseph`
  пытается собираться из C-исходников и падает без Microsoft Build Tools).
- Эфемериды в питон-эталоне: форк **`pysweph`** (не `pyswisseph`). Импорт тот же:
  `import swisseph as swe`. Математика Swiss Ephemeris сохранена полностью —
  логика расчётов идентична оригиналу.
- Зависимости питон-части: `pysweph`, `python-telegram-bot[job-queue]`
  (для старого бота), `python-dotenv`.
- Установка через `pip` в чистом `.venv` на 3.12; собранные wheel ставятся без
  компилятора.
- Консоль — **UTF-8** (постоянно): `PYTHONUTF8=1`, `PYTHONIOENCODING=utf-8`,
  `LANG=C.UTF-8` в User-окружении + профиль PowerShell + `~/.bashrc`. Скрипты
  могут НЕ городить `sys.stdout.reconfigure` — кириллица/°/глифы и так чистые.

> Эта история нужна, чтобы НЕ повторять ошибку с Python 3.14 и помнить про форк
> `pysweph`. Питон-часть здесь — только ЭТАЛОН для сверки (см. ниже), не движок
> приложения на Android.

## Жёсткие правила расчёта (НЕ нарушать)
- Только Swiss Ephemeris. Никаких самодельных формул эфемерид / расчётов «по памяти».
- Тропический зодиак (без сидерики/Джйотиш).
- Все расчёты в UTC; перевод в пояс — только при выводе.
- Аудит элонгаций: Меркурий <~28° от Солнца, Венера <~48° — иначе «расчёт сломан».
- Аспекты — по математической разнице градусов, не по линиям на картинке.
- Объекты (БАЗОВЫЕ, всегда вкл.): Солнце, Луна, Меркурий, Венера, Марс, Юпитер,
  Сатурн, Уран, Нептун, ИСТИННЫЕ узлы (Раху/Кету). Мажорные аспекты: ☌ ⚹ □ △ ☍.
  Орбис по умолчанию 1°, настраиваемый.
- ДОП. объекты (вкл./выкл. + догрузка) — пожелание астролога 2026-06-27. В скоупе:
  - из коробки (в бандле `seas_18.se1`): Хирон, Фолус, Церера, Паллада, Юнона,
    Веста, Лилит (ср. `12`, ист. `13`); считаются сразу;
  - далёкие карлики/TNO (Эрида `10000+136199`, Макемаке `+136472`, Хаумеа
    `+136108`, Седна `+90377` и пр.) — нужен отдельный файл `.se1` на объект,
    кладётся в `/sweph/astN/` (см. «Доп. объекты и догрузка» ниже).
  - UI: тумблер показа каждого доп. объекта; недостающие файлы — догрузить/докачать.

## Архитектурные решения
- Платформа: **web + Capacitor → APK** (см. ЗАДАНИЕ §1, развилка A).
  - РЕШЕНО (2026-06-27): остаёмся на **Capacitor**, НЕ переходим на ручной
    Kotlin-проект (ЗАДАНИЕ §10.2). Обоснование: Capacitor тоже даёт настоящий
    устанавливаемый APK с уведомлениями/хаптикой/защ. хранилищем — пожелание
    «нативное приложение» закрыто ПО РЕЗУЛЬТАТУ, при «минимуме кода» (§0, §8).
    Ручной Kotlin появляется только как часть фолбэка B2 (см. ниже).
- Движок на устройстве: РЕШИТЬ на шаге 1 (спайк) —
  - B1: Swiss Ephemeris **WASM** — пакет **swisseph-wasm** (см. статус ниже).
    На устройстве — режим **SWIEPH (полные файлы эфемерид)**, РЕШЕНО 2026-06-27:
    астролог работает с астероидами/TNO, а они БЕЗ файлов не считаются (Moshier их
    не умеет). Файлы уже в пакете (`swisseph.data`, ~12 МБ). Спайк-сверка делалась
    в Moshier (эталон pysweph без файлов); на устройстве SWIEPH — точнее, как
    astro.com. Это приоритет;
  - B2: **Chaquopy + pysweph** (если B1 не выходит) — питон-точный, но нативная сборка.
  - B3 (чисто-JS, НЕ Swiss Eph) — только с явного согласия владелицы.
- UI — **движок-независим**: расчёт за интерфейсом функций (ЗАДАНИЕ §2.3).
- Колесо неба и глифы — без генерации картинок (готовая SVG-либа + символьный шрифт).
- Сборка APK: РЕШЕНО (2026-06-27) — **локально** (Android Studio/Gradle):
  бесплатно и без лимитов. Автосборка по коммиту — опционально через
  **GitHub Actions** (публ. репо — безлимит; приватный — 2000 мин/мес, хватает
  с запасом). Раздача астрологу — sideload APK, Play Store/$25 не нужны.

> СТАТУС выбора движка (2026-06-27): **B1 ПОДТВЕРЖДЁН ПОЛНОСТЬЮ** (Capacitor +
> Swiss Eph WASM). Движок — пакет **swisseph-wasm** (emscripten-сборка swe

## Сверка с эталоном (обязательна перед UI)
Папка `engine_reference/` — работающий код на `pyswisseph/pysweph`. Любой новый
движок обязан давать ИДЕНТИЧНЫЙ результат: долготы до 0.01°, времена точных
аспектов до минуты. Контрольные 5–10 дат, сверить, записать результат сюда.

**РЕЗУЛЬТАТ (2026-06-27).** Спайк в `spike/`. `dump_reference.py` дампит эталон
(pysweph 2.10.03, фактически Moshier — файлов эфемерид нет) в `reference_dump.json`
(позиции, аспекты, затмения+Сарос). Затем три verify-скрипта гоняют движок
**swisseph-wasm** (swe 2.10.03, Moshier) через общий `engine.mjs` и сверяют:
`verify_wasm.mjs` (позиции), `verify_aspects.mjs` (времена аспектов),
`verify_eclipses.mjs` (затмения §10.1). Даты позиций: 2000-01-01, 2010-09-09,
2020-12-21, 2023-03-02, 2025-01-01, 2026-06-27. Итог: позиции до **0.0000″**,
аспекты ≤ **0.49 сек**, Сарос **10/10**. ✅ B1 подтверждён полностью.

Прогон: `cd spike && npm install && npm run verify`. Пересборка эталона —
`../.venv/Scripts/python.exe dump_reference.py` (через `npm run dump:ref`
путь к python в cmd не резолвится — звать python напрямую).

**Попутно исправлен баг ЭТАЛОНА** (`engine_reference/daily_aspects_astro.py`,
`_exact_time`): точное время не считалось для СОЕДИНЕНИЙ и ОППОЗИЦИЙ (там
`|sep|-target` лишь касается 0/180 и не меняет знак → было `exact_time=None` для
всех ☌ и ☍). Теперь для ☌ ищем ноль знаковой `angdiff(lon1,lon2)`, для ☍ —
`angdiff(delta,180)`. Проверено: Юпитер–Сатурн 2020-12-21 → 18:20:37 UTC
(=0°29′10″ Водолея). Оставшиеся `None` у ☌ — легитимные «близкие прохождения»
(мин. разница >0, точного 0° нет, напр. Нептун–Кету 0.25°).

**Окно орбиса (требование астролога).** Аспект — это ИНТЕРВАЛ, не «весь день».
В записи аспекта теперь `begin_time`/`end_time` (вход/выход орбиса) рядом с
`exact_time` — `_orb_distance`/`_orb_edge` в эталоне. Окно может выходить за
сутки: Юпитер–Сатурн ☌ 2020-12 — окно 12→30 дек (18 дней); Луна–Сатурн ⚹ —
вход 21 дек 21:32, выход 22 дек 01:33 (через полночь). UI обязан показывать
интервал, а не аспект «на весь день».

> ВЫБОР ПАКЕТА WASM (2026-06-27): взят **swisseph-wasm** (prolaxu), а НЕ
> @swisseph/browser@1.1.1. Причина: @swisseph отдаёт у затмений только времена
> (нет `*_eclipse_how`/`*_where` в экспортах) → §10.1 (Сарос) невыполним; плюс
> его dist-бандл сломан (CJS-куски в ESM, ссылки свёрнуты в `(void 0)` — жил на
> заплатке patch_vendor.mjs). swisseph-wasm — штатный emscripten-glue (заплаток
> НЕ нужно), экспортирует ПОЛНЫЙ низкоуровневый API (sol/lun eclipse
> when/where/how) → Сарос берём через `Module.ccall` (см. `spike/engine.mjs`).
> Бонус: в пакете есть и полные файлы эфемерид (`swisseph.data`, ~12 МБ) — на
> устройстве можно Moshier (без файла) ИЛИ полный SWIEPH (точнее, +12 МБ в APK,
> решить при сборке). Минусы: пакет молодой (v0.0.5, один автор) и GPL-3 — но это
> лишь сборка канонического C-исходника Swiss Eph, при нужде пересоберём сами.
> Сарос: серия=`attr[9]`, член=`attr[10]` из `*_eclipse_how`; для солнечных
> сперва `sol_eclipse_where` (точка максимума), иначе how вернёт 0.

## Доп. объекты и догрузка (астероиды/TNO) — проверено 2026-06-27
Режим SWIEPH. Код объекта: астероид = `SE_AST_OFFSET(10000) + номер`. Расчёт —
обычный `swe_calc_ut(jd, ipl, FLG_SWIEPH|FLG_SPEED)`.
- **В бандле `swisseph.data`** (смонтирован в `/sweph`): `sepl_18`, `semo_18`,
  `seas_18` + таблицы. Считаются СРАЗУ: Хирон(15), Фолус(16), Церера(17),
  Паллада(18), Юнона(19), Веста(20), Лилит ср.(12)/ист.(13). Спайк: ✅ все.
- **Чего нет в бандле:** индивидуальные файлы TNO. Имя файла даёт сама библиотека
  в тексте ошибки: Эрида → `s136199.se1` (длинный, 3000 BCE–2999 CE) или
  `s136199s.se1` (короткий 1500–2100); номера ≥100000 начинаются с `s`, <100000 —
  с `se`. Файл ОБЪЕКТА кладётся в подпапку `astN` (N=номер/1000): Эрида →
  `/sweph/ast136/s136199.se1`.
- **Догрузка во время работы РАБОТАЕТ** (проверено): у emscripten-модуля доступна
  виртуальная ФС — `swe.SweModule.FS.writeFile('/sweph/astN/<файл>.se1', bytes)`,
  затем `calc_ut` сразу видит объект. Спайк `spike/probe_extra_objects.mjs`:
  Церера на 1000 г. падала «нет `seas_06.se1`» → после записи файла в ФС
  посчиталась. Тот же путь для Эриды/Макемаке.
- **РЕШЕНО 2026-06-27: набор астролога ВШИТ в APK** (оффлайн-первый; ~15–20 КБ на
  объект, Эрида 171 КБ — итого ~0.3 МБ, мелочь). Астролог прислала 19 файлов
  (папка `sw/`), разложены в `ephe/astN/`: Хаос(19521), Варуна(20000),
  Иксион(28978), Хуйя(38628), Квавар(50000), 2002 UX25(55637), 2003 VS2(84922),
  Седна(90377), Оркус(90482), 2004 GV9(90568), Салация(120347), Хаумеа(136108),
  Эрида(136199), Макемаке(136472), Варда(174567), 2003 AZ84(208996),
  Гонгун(225088), 2002 MS4(307261), Лелеакухонуа(541132). Все, кроме Эриды, —
  «короткие» (1500–2100). При сборке копируются в ассеты, на старте пишутся в
  `Module.FS`. Имена/коды — data-driven (скан `ephe/ast*/`), см. `dump_swieph.py`.
- **Источник файлов TNO:** Dropbox Алоиса (`long_ast/astN/` длинные,
  `all_ast/astN/` короткие) — но это JS-страница, СКРИПТОМ (curl) НЕ качается
  (отдаёт HTML-обёртку, не бинарь; настоящий файл начинается со слова `SWISSEPH`).
  Поэтому файлы взяты ГОТОВЫЕ у астролога. Ядро (`sepl_18/semo_18/seas_18`) — на
  GitHub `aloistr/swisseph/ephe` (лежит в `ephe/` для питон-эталона; в APK не
  нужно — оно внутри `swisseph.data`).

**СВЕРКА SWIEPH↔SWIEPH (2026-06-27).** `spike/dump_swieph.py` (pysweph с файлами
из `ephe/`) ↔ `spike/verify_swieph.mjs` (swisseph-wasm; те же файлы пишутся в его
ФС → читают идентичные данные). **38 объектов** (база + Хирон/Фолус/Церера/
Паллада/Юнона/Веста + Лилит ср./ист. + 19 карликов/TNO астролога + Кету) × 6 дат
= **228 значений**, 0 ошибок: макс. ошибка долготы **0.069″** (≈2·10⁻⁵°, худший —
Марс; сами TNO ещё точнее), скорости 4·10⁻⁷°/сут — внутри допуска с запасом
(остаток — нюанс под-версии файлов/ΔT, для астрологии незаметен). ✅ Прогон:
`cd spike && ../.venv/Scripts/python.exe dump_swieph.py && npm run verify:swieph`.

Пример «почему числа зависят от времени» (частый вопрос): Великое соединение
Юпитер–Сатурн 2020-12-21. На 12:00 UTC Сатурн 0°27′ Водолея, но ТОЧНОЕ
соединение — в 18:20:36 UTC, и тогда обе планеты ровно в **0°29′10″ Водолея**
(как в астропроцессорах). Снимок на полдень ≠ момент точного аспекта.

## UI-решения
- **Хранение данных:** файл на диске через File System Access API (`lib/fileStore.ts`,
  `lib/db.ts`) — переживает чистку кэша; + экспорт/импорт читаемого JSON. На
  Android позже — Capacitor Filesystem в выбранной папке (§10.6), контракт тот же.
- **Колесо неба (§3.8):** РЕШЕНО 2026-06-27 — БЕЗ домов (транзитный круг: зодиак +
  планеты + линии аспектов). Дома в ТЗ §3.8 нет; они требуют места/времени и
  крутятся за сутки. Сделан СВОЙ лёгкий SVG (`ui/Wheel.svelte`) на данных движка:
  готовая `@astrodraw/astrochart` не подошла (её radix всегда рисует дома/куспиды).
  Если позже захотят дома — вернуть astrochart ИЛИ свои куспиды через `swe_houses`
  (движок умеет) + ввод места + система домов.
- **Иконки/глифы:** Claude НЕ генерирует растровые картинки (vision = только ввод).
  Глифы — Unicode/SVG/символьный шрифт (§5, §8), не генератор. Иллюстрации богов
  (архетипы, раунд 3) — брать из открытых источников ИЛИ один раз внешним
  image-генератором, вшивать как оффлайн-ассеты (не за токены).
- **Символы знаков:** SVG-пути из Tabler Icons (MIT) в `ui/signIcons.ts` (формы
  Овен..Рыбы). Стиль выбирается в настройках (`Settings.signStyle`): серебро /
  золото+свечение / по стихиям / переливание (аним. градиент) / радуга — пикер в
  `DataPanel`, рендер в `ui/Wheel.svelte` (SVG-градиенты + filter glow). Это 5

## Раунд 4 — отзывы астролога (2026-06-30, реализовано)

- **Нижняя панель** (📅 Дата · 📓 Журнал · 📚 Библиотека · 💬 Чат · ⚙ Настройки),

- **OTA — живые обновления (Capgo, бесшовно):** `@capgo/capacitor-updater` (peer

## Раунд 5 — отзывы астролога (2026-07-01, реализовано)
Проверено `npm run check` (0/0) + `node test/parity.mjs` + `npm run build` +
`npx cap sync android` (5 плагинов). ⚠️ Добавлены НАТИВНЫЕ плагины → нужен
СВЕЖИЙ APK (OTA-веб их не доставит); старый APK получит только веб-правки.
- **ГЛАВНОЕ — данные не сохранялись после перезапуска** (заметки, архетипы, пояс,
  время уведомлений). Причина: на Android полагались только на `localStorage`
  WebView (File System Access API там нет), а он не переживает перезапуск/смену
  origin при OTA. ФИКС: durable-стор `@capacitor/preferences` (нативный
  SharedPreferences) в `lib/db.ts` — `scheduleNativeSave()` пишет, `hydrate()` на
  старте поднимает (await до UI, см. `App.onMount`). localStorage остался быстрым
  кэшем. Это же чинит «время уведомлений не сохраняется».
- **Архетипы ВШИТЫ в приложение** (`lib/lore.ts` `PLANET_LORE`/`defaultArchetypes`)
  — базовый набор всегда есть, `db.hydrate()` досевает недостающие, не затирая
  правки астролога. Решение по просьбе: «жёсткая привязка» архетипов.
- **Обучалка (новое направление — приложение-учебник):**
  - приветствие первого запуска (`ui/Welcome.svelte`, текст `WELCOME` в `lore.ts`):
    какой школы держимся (западная транзитная, тропик, Swiss Eph) и правила.
    Флаг `Settings.seenWelcome`; повторно — Настройки → Обучение;
  - тык по символу в КОЛЕСЕ (планета ИЛИ линия аспекта) → разбор
    (`ui/InfoSheet.svelte` + `WheelInfo`, хит-зоны в `ui/Wheel.svelte`,
    проброс `oninfo` через `DayScreen`). Контент — `PLANET_LORE`/`ASPECT_LORE`.
- **Попап аспекта** (`ui/InterpretationSheet.svelte`): по просьбе астролога УБРАН
  блок «Трактовка»; осталась заметка + кнопка «Обсудить аспект с Claude» по
  заложенным архетипам (`ondiscuss` → `App.openChat(seed)` → `ChatSheet seed`
  авто-отправляет). Так же кнопка «Обсудить» в `InfoSheet`. (Библиотека
  «Трактовки» осталась как read-only список прежних записей.)
- **#13 Уведомления — нативно** (`@capacitor/local-notifications`, `lib/notifications.ts`):
  ежедневная сводка (повтор по времени) + момент точного аспекта планета–планета
  (одноразовые на 10 дней, без Луны). `syncNotifications(engine,settings,tz)` зовётся
  из `App` на старте и при смене настроек. Тумблер «в момент аспекта» в `DataPanel`.
- **#5/#12 экспорт/импорт на Android** (`lib/backup.ts`): экспорт — `@capacitor/filesystem`
  (Cache) + `@capacitor/share`; импорт — общий `<input type=file>` (работает в WebView).
- **#7 «Подгрузить миф (Claude)»** в `ArchetypesSheet` (`chat.loadDeityMyth`): краткий
  пересказ + параллели из др. культур, без «графы рассуждений».
- **Скорость прокрутки (жалоба «подтормаживает»):** убран `backdrop-filter: blur`
  с потоковых `.glass`-карточек (их в ленте десятки — пересчёт размытия на каждом
  кадре скролла); blur оставлен только на оверлеях через класс `.frost` (шапка,
