# DESIGN_BRIEF — оформление Astra (внешняя дизайн-сессия)

Задача: сделать **один** цельный вид интерфейса. Работать в ветке
`design/neon-stardust`, поверх актуального `main`. Это ТОЛЬКО оформление —
функциональность и расчёты не трогать.

## Цель темы (в трёх словах)
**Тёмная база + неоновые светящиеся линии + летающие светлые пылинки-звёздочки.**

## Что уже есть (текущее состояние)
Приложение уже тёмное и «космическое»: тёмно-синий фон, золотые глифы зодиака,
планеты в цветных «монетках», тонкие линии аспектов, стеклянные карточки (`.glass`).
Экраны: лента дней с шапкой-датой, колесо неба (SVG), карточка Луны, панель позиций,
нижний таб-бар (Дата · Журнал · Библиотека · Чат · Настройки).

Тема выбирается в `Settings.theme` (`cosmos`/`dawn`/`auto`) — сейчас можно свести к
одному виду (тёмному) как основному.

## Что сделать
1. **Неон.** Линии аспектов в колесе и ключевые штрихи/границы — со свечением
   (`filter: drop-shadow` для SVG, `box-shadow`/`text-shadow` для DOM). Аккуратный
   неоновый акцент (циан/маджента/фиолет — подобрать), не «кислота». Глифы могут
   получить лёгкий glow.
2. **Летающие звёздочки-пылинки.** Анимированный слой светлых частиц, медленно
   дрейфующих (parallax/мерцание). Должен быть ЛЁГКИМ (см. производительность ниже) —
   один фоновый слой, не десятки анимированных нод в ленте.
3. **Верхняя панель телефона (см. скриншот-жалобу).** Контент «заходит» под системный
   статус-бар с часами — шапка-дата прижата к часам без отступа. Нужно уважать
   `env(safe-area-inset-top)` на шапке/`main` (снизу `safe-area-inset-bottom` уже
   учтён) и сделать переход к статус-бару эстетичным. При желании — настроить
   `@capacitor/status-bar` (overlay + цвет иконок под тёмный фон).

## Жёсткие рамки (НЕ нарушать)
- **Не трогать `app/src/engine/*`** и контракт движка — только вид.
- **Глифы/иконки — без растровой генерации** (Unicode/SVG/шрифт). Символы знаков —
  SVG-пути Tabler Icons в `ui/signIcons.ts`.
- **Svelte 5:** db-коллекции возвращают ту же ссылку массива — в компонентах
  `db.x.all().slice()`, иначе `$state/$derived` не обновятся.
- **Производительность (была жалоба «подтормаживает»):** `backdrop-filter: blur`
  ТОЛЬКО на оверлеях через класс `.frost` (шапка, таб-бар, шторки) — НЕ на потоковых
  `.glass`-карточках ленты. Космос/звёзды — один неподвижный `fixed`-слой, без
  `background-attachment: fixed`.
- **Capacitor WebView** (системный Chromium): современный CSS (`color-mix`,
  `env()`, `filter`) ок, но проверять на реальном Android, а не только в десктоп-браузере.
- Оффлайн-первый: ассеты вшивать локально, без внешних CDN/шрифтов по сети.

## Ключевые файлы для темы
- `app/src/app.css` — палитра (CSS-переменные), неподвижный звёздный слой, классы
  `.frost`/`.glass`, `html[data-theme]`, `html[data-font='large']`.
- `app/src/ui/Wheel.svelte` — SVG-колесо: зодиак, глифы, линии аспектов, градиенты,
  filter-glow. Здесь живёт основной «неон».
- `app/src/ui/signIcons.ts` + `Settings.signStyle` — 5 цвето-стилей символов знаков
  (серебро/золото+glow/по стихиям/переливание/радуга).
- Шапка и таб-бар — в `app/src/App.svelte` (тут же `env(safe-area-inset-*)`).
- Шторки — `<section class="sheet glass frost">` в `app/src/ui/*Sheet.svelte`.

## Как запускать (быстрый цикл превью)
```
cd app
npm install
npm run dev      # Vite dev-сервер, превью в браузере — самый быстрый цикл по CSS
npm run check    # типы (должно быть 0 ошибок) перед коммитом
```
Сборка APK — через CI (push), локально не обязательна для дизайна.
