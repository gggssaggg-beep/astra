# CLAUDE.md — память проекта для Claude Code

Локальное Android-приложение для астролога (**Astra**). Оффлайн-первое, русский
интерфейс. Этот файл — только короткие правила, которые надо держать в голове
всегда. Тяжёлые детали вынесены (грузить по надобности, не в каждый контекст):
- `ЗАДАНИЕ.md` — полное ТЗ (разделы 0–10).
- `docs/HISTORY.md` — раунды правок, сверки движка, детали астероидов/TNO,
  обоснования решений.
- `DESIGN_BRIEF.md` — текущая задача на оформление (тема интерфейса).

## Что строим (одна строка)
Оффлайн «небо по дням»: аспекты, колесо транзитов, заметки/журнал, напоминания,
библиотека трактовок/архетипов, чат-трактовки на своём ключе Claude.

## Окружение (зафиксировано)
- ОС разработки — **Windows**. Консоль UTF-8 уже настроена (скрипты НЕ городят
  `sys.stdout.reconfigure` — кириллица/°/глифы чистые).
- Питон-эталон: **Python 3.12** в `.venv` (НЕ 3.14 — там `pyswisseph` требует
  Build Tools и падает). Эфемериды — форк **`pysweph`** (импорт `import swisseph
  as swe`). Питон-часть — ТОЛЬКО эталон для сверки, не движок приложения.

## Жёсткие правила расчёта (НЕ нарушать)
- Только Swiss Ephemeris. Никаких самодельных формул эфемерид «по памяти».
- Тропический зодиак (без сидерики/Джйотиш). Все расчёты в UTC; в пояс — при выводе.
- Аспекты — по математической разнице градусов, не по линиям на картинке. Мажорные:
  ☌ ⚹ □ △ ☍. Орбис по умолчанию 1°, настраиваемый (индивидуально по объекту; для
  пары — БОЛЬШИЙ из двух).
- **Аспект — это ИНТЕРВАЛ (вход → точно → выход орбиса), не «весь день»**; окно
  может переходить через полночь. UI обязан показывать интервал.
- Позиции — в знаке+градусе (0°29′ Водолея), не сырой долготой; сверять на МОМЕНТ
  точного аспекта, не на полдень.
- Аудит элонгаций: Меркурий <~28° от Солнца, Венера <~48° — иначе «расчёт сломан».
- Объекты (БАЗОВЫЕ, всегда вкл.): Солнце, Луна, Меркурий, Венера, Марс, Юпитер,
  Сатурн, Уран, Нептун, ИСТИННЫЕ узлы (Раху/Кету). Порядок аспектов — по удалённости
  от Солнца (Солнце перед Меркурием/Венерой).
- ДОП. объекты (тумблер вкл./выкл.): Хирон/Фолус/Церера/Паллада/Юнона/Веста/Лилит
  (из бандла) + карлики/TNO астролога (вшиты в `ephe/astN/`, data-driven скан).
  Коды/имена файлов/догрузка — см. `docs/HISTORY.md`.

## Архитектура (решено 2026-06-27)
- Платформа: **web + Capacitor → APK** (не ручной Kotlin). Раздача — sideload,
  без Play Store.
- Движок на устройстве: **B1 — Swiss Eph WASM, пакет `swisseph-wasm`**, режим
  **SWIEPH** (полные файлы эфемерид — нужны для астероидов/TNO). Подтверждён
  сверкой с питон-эталоном (детали и цифры — `docs/HISTORY.md`). B2 (Chaquopy +
  pysweph) — фолбэк, не нужен. B3 (чисто-JS) — только с явного согласия владелицы.
- **UI движок-независим** (расчёт спрятан за интерфейсом функций, ЗАДАНИЕ §2.3).
  Колесо/глифы — без генерации картинок (SVG + символьный шрифт).
- Сборка APK: локально (Android Studio/Gradle) ИЛИ CI (GitHub Actions,
  `build-apk.yml`). Подпись — фиксированным `app/android/app/astra-debug.keystore`
  (стабильная подпись → обновление ставится ПОВЕРХ без потери данных; ключ вшит в
  репо, генерится в CI один раз — это не секрет, debug/sideload-ключ).
- **OTA (Capgo, `@capgo/capacitor-updater`, ручной режим):** веб-правки едут живым
  обновлением без переустановки; НАТИВНЫЕ плагины/keystore — только новый APK.

## Хранение и реактивность (частые грабли)
- Данные — `lib/db.ts`: localStorage (быстрый кэш) + на устройстве durable
  `@capacitor/preferences` (`hydrate()` на старте до UI, `scheduleNativeSave()`) +
  опц. файл на диске (File System Access) с экспортом/импортом читаемого JSON
  (`lib/backup.ts`). На Android localStorage НЕ durable — источник правды Preferences.
- Формат сохранения — обёртка `{app,schema,savedAt,data}`; читатели берут `.data`
  через `payload()` (иначе коллекции поднимаются ПУСТЫМИ — был баг «не сохраняется»).
- **Svelte 5:** `db.x.all()` возвращает ТУ ЖЕ ссылку массива (мутируется на месте);
  в компонентах брать `db.x.all().slice()`, иначе `$state/$derived` не обновятся.

## Конвенции
- Оффлайн-первый: ядро без сети. Сеть только для чата трактовок (§3.6). ИИ НЕ
  участвует в расчётах — только чат по запросу, на ключе пользователя.
- Готовые библиотеки вперёд (колесо, свайпы, календарь, уведомления, хранение);
  велосипеды обосновывать. Минимум зависимостей и кода.

## Безопасность
- Ключ Anthropic — ТОЛЬКО в защищённом хранилище устройства (`lib/secret.ts`), НЕ
  в коде, НЕ в гите, НЕ в файле данных. `.env`, ключи, секреты — в `.gitignore`.

## UI-решения (кратко)
- Колесо неба — БЕЗ домов (транзитный круг: зодиак + планеты + линии аспектов),
  свой лёгкий SVG `ui/Wheel.svelte`. Дома — только если попросят (`swe_houses` +
  ввод места + система домов).
- Глифы — Unicode/SVG/шрифт (Claude НЕ генерит растровые картинки — vision только
  на ввод). Символы знаков — Tabler Icons (MIT) в `ui/signIcons.ts`; 5 цвето-стилей
  (`Settings.signStyle`).
- Нижняя панель: 📅 Дата · 📓 Журнал · 📚 Библиотека · 💬 Чат · ⚙ Настройки. Любая
  шторка закрывается свайпом-вниз (`lib/sheet.ts`, уважает внутренний скролл).
- Архетипы богов — на каждый объект (`lib/lore.ts`, редактируемые, вшиты). Чат по
  аспекту автосохраняется в журнал (тег объектов+сигнатуры → «Похожие прошлые»).

## Где что
- `engine_reference/` — питон-эталон для сверки (ЗАДАНИЕ §9).
- `spike/` — сверки движка (Moshier, SWIEPH↔SWIEPH, затмения/Сарос, доп. объекты).
- `ephe/` — файлы эфемерид SWIEPH (ядро + карлики/TNO); см. `ephe/README.md`.
- `app/` — приложение (Vite+Svelte+TS+Capacitor). `app/src/engine/` — движок-
  независимое ядро (порт эталона), `app/test/` — парити, `app/src/lib/` — стор/
  формат/чат/уведомления/OTA/секрет, `app/src/ui/` — экраны и шторки.
